# unified-portal-frontend 服务关系梳理


## 项目概述


unified-portal-frontend 是 CVTE 统一门户前端项目，作为企业内部各业务系统的统一入口网关，聚合了身份认证、业务应用、监控追踪、消息通知等多个子系统。

## 后台服务清单


### 1. 身份认证服务


| 服务名 | 域名 | 根路径 | 说明 |
| --- | --- | --- | --- |
| portal-auth | `idm-admin.gz.cvte.cn` | `/authentication-api` | 认证 API，负责登录、Token 签发 |
| portal-account | `idm-admin.gz.cvte.cn` | `/account-api` | 账号管理 API，用户增删改查 |
| portal-author | `idm-admin.gz.cvte.cn` | `/authorization-api` | 授权 API，权限与角色管理 |
| SSO/CAS | `login.cvte.com` | `/` | 统一单点登录，基于 CAS 协议 |
| IAC | `itapis.cvte.com/iac` | `/` | 内部 API 中心，Token 管理 |


**调用链路**: 用户访问门户 → Koa 中间件检查 Session → 未登录则跳转 SSO/CAS → CAS 回调验证 → 写入 Redis Session → 前端获取用户信息通过 portal-account API

### 3. 第三方平台集成


#### 3.1 短信服务 (SMS)


**服务域名**: `itapis.cvte.com`
**代码位置**: `server/services/login/index.ts`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |
| 发送短信验证码 | POST | `https://itapis.cvte.com/verification-code/sms/send` | `{ phone }` | 密码重置/短信登录验证码 |
| 校验短信验证码 | POST | `https://itapis.cvte.com/verification-code/sms/verify` | `{ phone, verificationCode }` | 校验用户输入的验证码 |


**认证方式**: Header `access-token` 携带 IAC Token（通过 `[@cvte](https://github.com/cvte)/ciac` SDK 获取）


#### 3.2 邮件服务 (Email)


**服务域名**: `itapis.cvte.com/wopush-platform`
**代码位置**: `server/services/login/index.ts`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |
| 发送邮件（外部） | POST | `https://itapis.cvte.com/wopush-platform/push/external` | `{ canReply, notifyWay: '0010', content, title, toUser }` | 密码重置邮件 |
| 发送通知（内部） | POST | `https://itapis.cvte.com/wopush-platform/push/internal` | `{ canReply, notifyWay, content, title, config, toUser }` | 企业微信内部通知 |


**参数说明**:


- `notifyWay`: `'0010'` 表示邮件渠道
- `toUser`: JSON 字符串，格式 `[{"email":"[xxx@cvte.com](mailto:xxx@cvte.com)"}]`
- `title`: 邮件标题，如 `'[CVTE] 确认重置密码'`
- `content`: HTML 格式邮件内容，包含重置链接


**认证方式**: Header `access-token` 携带 IAC Token


#### 3.3 ITGW 统一网关（SMS/邮件/企微的新版通道）


**服务域名**: `itgw.cvte.com`
**代码位置**: `server/services/login/index.ts`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |
| 内部推送 | POST | `https://itgw.cvte.com/env-101/por-6/common/wopush-platform/push/internal` | 同 WOPush internal | 新版 API 平台发送企微/短信/邮件 |
| 外部推送 | POST | `https://itgw.cvte.com/env-101/por-6/common/wopush-platform/push/external` | 同 WOPush external | 新版 API 平台发送邮件 |


**认证方式**: Header `x-iac-token` + `apiKey`


#### 3.4 企业微信 (WMP)


**服务域名**: `wmp.cvte.com`
**WMP ID**: `d72afa0c-f3c7-40c8-b9cd-ede07599a0cc`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |
| 扫码登录页 | GET | `https://wmp.cvte.com/{wmpId}/qy_scan_login` | `appId, token` | 发起企微扫码登录 |
| 扫码票据验证 | GET | `https://wmp.cvte.com/{wmpId}/qy_scan_ticket_verify` | `appId, ticket` | 验证扫码结果 |
| OAuth 获取用户 | GET | `https://wmp.cvte.com/apis/{wmpId}/qy_oauth_get_user` | `code` | 通过 OAuth code 获取用户信息 |
| OAuth 跳转 | GET | `https://wmp.cvte.com/{wmpId}/oauth?url={redirectUrl}` | `url` | 企微客户端 OAuth 登录入口 |


**前端路由**:


- `GET /wmp_oauth_init` — 企微 OAuth 回调，参数: `code, appId, next`


**代码位置**: `server/routes/pages/index.ts`, `server/common/utils.ts`, `server/routes/apis/controller/login/index.ts`


#### 3.5 简道云 (JDY) SSO


**服务域名**: `jdy.cvte.com` / `tjdy.cvte.com`
**集成方式**: JWT 签发，门户作为 IdP 向简道云签发 Token


| 前端路由 | Method | 参数 | 用途 |
| --- | --- | --- | --- |
| `/{sysPrefix}/custom/auth/wxId/:id` | GET | `request (JWT), state` | 以企微 ID 为用户名的 SSO |
| `/{sysPrefix}/custom/auth/account/:id` | GET | `request (JWT), state` | 以域账号为用户名的 SSO |


**SSO 流程**:


- 简道云携带 JWT request token 跳转到门户
- 门户验证用户登录态（cookie userInfo）
- 门户使用 corpId 对应的 secret 签发新 JWT
- 重定向到简道云 ACS URL，携带 response JWT


**代码位置**: `server/routes/pages/controller/sso/jdyAuth/index.ts`


#### 3.6 飞书 (Feishu/Lark) SSO


**服务域名**: `open.feishu.cn`
**App ID**: `cli_a9448d123438dcd5`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |

**前端路由**: `GET /{sysPrefix}_papi/sso/feishu/oauth`


**代码位置**: `server/routes/apis/controller/sso/index.ts`


#### 3.7 智汇云 (Zhihuiya) SSO


**服务域名**: `passport.zhihuiya.com`


| 接口 | Method | URL | 参数 | 用途 |
| --- | --- | --- | --- | --- |
| 获取 SSO Token | GET | `https://passport.zhihuiya.com/public/sso/token` | `identify_key (RSA 加密邮箱)` | 获取智汇云登录 Token |


**SSO 流程**: 门户获取 Token 后重定向到 `https://passport.zhihuiya.com/public/sso/login?token={token}&redirect_uri=https://analytics.zhihuiya.com`

**前端路由**: `GET /spec/zhihuiya`

## API 路由规则


前端请求通过 Koa 代理层转发到后端服务，路由格式：


```
/{serviceName}_{suffix}/*
```


| 后缀 | 含义 | 是否鉴权 |
| --- | --- | --- |
| `_api` | 标准 API | 需要认证 |
| `_fapi` | 前端专用 API | 需要认证 |
| `_papi` | 公开 API | 无需认证 |


示例：`/portal-auth_api/v1/token` → 代理到 `idm-admin.gz.cvte.cn/authentication-api/v1/token`
