# 从门户跳转

## 用户通过门户跳转到目标业务系统


-

目标业务系统提供 SSO 初始化接口，用户跳转至该接口，携带 ticket，ticket 有效期为 120 秒

-

目标业务系统获取到 ticket，再加上门户派发的 appId，调用门户提供的 ticket 校验接口，换取用户信息和 jwt

-

目标业务系统获取到用户信息和 jwt 之后，自行设置 cookie，保存用户会话


## 对接流程：(以测试环境举例)


1、在门户中注册你的业务系统，发送下面这段文本给@黄威鸿进行注册(需要说明注册的是测试环境还是正式环境)，注册成功后，会返回门户的appId给你：


```
 {
      appName: 'xxxx',   //系统名称

      appDesc: 'xxxxx'   //系统描述

      domain: 'httpxxxxxx'     // 系统地址

      root: 'xxxxxx'      // 需要让什么用户可以进行访问，例如：全员

 }
```


2、当检测到token过期时，需要重定向到[https://op-fat.cvte.com/login?redirect=URICode](https://op-fat.cvte.com/login?redirect=URICode)( [https://op-fat.cvte.com/portal/app_oauth?appId=xxxxxxxxxx&redirect=URICode(业务系统地址)](https://op-fat.cvte.com/portal/app_oauth?appId=xxxxxxxxxx&redirect=URICode(业务系统地址)) );


注意点：


>


1、上述中提到的URICode指的是要对这一段地址编码，用encodeURIComponent进行编码


>


2、实际中用到的重定向地址是这样的：
 [https://op-fat.cvte.com/login?redirect=%22%2Fportal%2Fapp_oauth%3Fredirect%3Dhttps%253A%252F%252Fop-admin-fat.gz.cvte.cn%252Foauth%253Fn%253D%25252Fpages%25252Fhome%2525253F%26appId%3Dxxxxxxxxxx%26type%3Dop%22](https://op-fat.cvte.com/login?redirect=%22%2Fportal%2Fapp_oauth%3Fredirect%3Dhttps%253A%252F%252Fop-admin-fat.gz.cvte.cn%252Foauth%253Fn%253D%25252Fpages%25252Fhome%2525253F%26appId%3Dxxxxxxxxxx%26type%3Dop%22) [https://op-fat.cvte.com/login?redirect=%22%2Fportal%2Fapp_oauth%3Fredirect%3Dhttps%253A%252F%252Fop-admin-fat.gz.cvte.cn%252Foauth%253Fn%253D%25252Fpages%25252Fhome%2525253F%26appId%3Dxxxxxxxxxx%26type%3Dop%22](https://op-fat.cvte.com/login?redirect=%22%2Fportal%2Fapp_oauth%3Fredirect%3Dhttps%253A%252F%252Fop-admin-fat.gz.cvte.cn%252Foauth%253Fn%253D%25252Fpages%25252Fhome%2525253F%26appId%3Dxxxxxxxxxx%26type%3Dop%22)


3、登录时，如果出现4000002的错误，就是角色未授权，出现4000004就是域名和门户的appId未绑定，可以联系（正式环境提单给IT客服、测试环境联系@黄威鸿）进行解决。登录成功后系统会重定向你刚刚填写的redirect地址【业务系统地址】，参数是一个ticket，ticket的过期时间是120S。例如：[http://www.yewu.com?ticket=xxxxxxxxxx](http://www.yewu.com?ticket=xxxxxxxxxx)


4、跳转到业务系统后，需要调用自己业务系统的一个oauth接口，把ticket、appId、x-iac-token传递过去。在oauth接口中调用门户提供的ticket_validate，拿到用户信息，业务自行设置cookie。保存用户会话。


## 门户接口


>


本系统已对接【[接口授权中心](https://kb.cvte.com/pages/viewpage.action?pageId=45811964)】，使用前请查。


host 分为测试和生产环境*（由于 4A 只有两个环境，门户做账号对接，所以只能提供两个环境，业务的 fat 和 uat 环境均对接门户的测试环境）*


测试：op-fat.cvte.com
生产：home.cvte.com


#### ticket 校验接口


**请求方式**：**GET**
**请求地址**：/apis/v1/ticket_validate
**参数说明**：


| 参数名 | 类型 | 含义 | 特殊格式说明 |
| --- | --- | --- | --- |
| x-iac-token | header | 【接口授权中心】中获取的 token | 无 |
| appId | query | 系统标识，由门户派发 | 无 |
| ticket | query | 身份凭证，由门户生成 | 无 |


**返回码说明**：


| status | message |
| --- | --- |
| 4000000 | iac 校验失败 |
| 4000001 | appId 和 ticket 必须提供 |
| 4000002 | ticket 无效或已过期 |
| 4000003 | appId 与 ticket 不匹配 |
| 5000001 | ticket 校验出错 |
| 5000002 | 获取用户信息失败 |
| 5000003 | 获取 jwt 失败 |

#### 登出接口：


**请求方式**：**GET**
**请求地址**：/portal/logout
**参数说明**：


| 参数名 | 类型 | 含义 | 特殊格式说明 |
| --- | --- | --- | --- |
| appId | query | 系统标识，由门户派发 | 无 |
| redirect | query | 校验用户后重定向的地址，由业务系统自定义 | 请对 query 参数做两层 encode，避免 query 参数中的 query 参数丢失（有点拗口，理解一下即可）如：[http://xx.cvte.com/oauth_init?next=/home?role=1](http://xx.cvte.com/oauth_init?next=/home?role=1) 则需要先对 /home 做一层 encode，为 %2fhome%3frole%3d1然后再对整个地址做 encode，为 http%3a%2f%2fxx.cvte.com%2foauth_init%3fnext%3d%252fhome%253frole%253d1 |


#### 目标系统接口


**请求方式**： 跳转（GET）
**请求地址**：业务系统自定义
**参数说明**：


| 参数名 | 类型 | 含义 | 特殊格式说明 |
| --- | --- | --- | --- |
| ticket | query | 身份凭证，由门户生成 | 无 |
