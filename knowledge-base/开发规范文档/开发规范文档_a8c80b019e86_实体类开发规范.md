# 实体类开发规范

## 1 方法命名规范


| 规则 | 说明 |
| --- | --- |
| 业务自定义方法不允许以set get作为开头，除非属性本身的set get方法 | JSON转换会报错，找不到属性或者执行失败 |
| get方法推荐使用got query match替换 |  |
| set方法推荐使用deal init替换 |  |


## 2 继承类属性赋值


| 规则 | 说明 |
| --- | --- |
| 属性赋值使用set方法不直接赋值 | 避免同名属性无法覆盖问题 |
| 属性取值使用get方法不直接引用 | 避免同名属性存在父子属性问题 |


## 2 属性命名


| 规则 | 说明 |
| --- | --- |
| 属性命名不允许有数字 | 避免驼峰下划线转换异常 |
| 属性取值使用get方法不直接引用 | 避免同名属性存在父子属性问题 |
