# 数据库设计规范

## 1 表规范


| 规范 | 说明 | 备注 |
| --- | --- | --- |
| 表名不能带数字 | 在属性字段驼峰下划线转换会有问题 |  |
| 表必须要带注释说明 | 说明表的作用，方便查找问题和责任人 |  |


## 2 字段规范


包括：命名 设置相关规范内容


#### 2.1 字段命名规范


| 规范 | 说明 | 备注 |
| --- | --- | --- |
| 不建议带数字 | 在属性字段驼峰下划线转换会有问题 |  |
| 必须要带有注释说明 | 避免不知道字段含义 |  |


#### 2.2 字段设置规范


| 规范 | 说明 | 备注 |
| --- | --- | --- |
| 不建议使用默认值 | 数据库克隆、迁移情况下会丢失，导致程序异常 | 推荐程序设置默认值，不依赖数据库 |
| 日期类型注意不同数据库区别 | pg使用TIMESTAMP ; Oracle使用DATE |  |
| 主键长度为32 | 不推荐使用带-36位的UUID |  |
| pg一张表中VARCHAR的总长度不能超过1.3万 | 默认数据库有总长度限制 | 推荐改为大文本 |
