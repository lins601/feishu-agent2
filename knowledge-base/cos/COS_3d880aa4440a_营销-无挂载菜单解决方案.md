# 营销-无挂载菜单解决方案

## 营销碰到的问题


- 多个内部路由同时匹配一个菜单载体，多个页面上报同一个菜单，导致性能统计不准
- 看板-功能视图没有展示对应的内部路由，无法展示内部路由页面的性能


## 解决方案


### 接入端


- 设置内部路由上报规则：pageCpde-{内部路由}
- 内部路由设置路由名称
- 构建打包时统计菜单和内部路由，组成菜单信息，内部路由code跟上报规则一致，上级parentId为菜单id，上报后台。后台提供一个菜单接口给性能监控


## 实现

#### 3. 日志上报


实现方式：原内部路由组件上，代理一层组件。替换原组件


```
const SetRouterWrap = (props) => {
  const { pageCode: pageCodeProps, lifecycle, url } = props;

  const pageCode = pageCodeProps ?? url?.split('/')?.pop() ?? '';

  let unlisten;
  const setContext = (location, action) => {
    const realPageCode = `${pageCode}@${location.pathname}`;
    if (action === 'PUSH') {
      setLocalMenuContext({
        menuId: undefined,
        menuName: undefined,
        menuCode: realPageCode,
      });
      bizPFRecorder.startTransaction({
        op: BIZ_TRANSACTION_OP.DETAIL_REFRESH,
        pageCode: realPageCode,
      });
    }
  };

  const mergelifecycle = {
    willUnmount: () => {
      lifecycle?.willUnmount?.();
      unlisten && unlisten();
      unlisten = null;
    },
    didMount: () => {
      lifecycle?.didMount?.();
      const self = window as any;
      if (self.contentHistory) {
        if (self.contentHistory?.location?.pathname) {
          setLocalMenuContext({
            menuId: undefined,
            menuName: undefined,
            menuCode: `${pageCode}@${self.contentHistory.location.pathname}`,
          });
          bizPFRecorder.startTransaction({
            op: BIZ_TRANSACTION_OP.DETAIL_REFRESH,
            pageCode: `${pageCode}@${self.contentHistory.location.pathname}`,
          });
        }
        unlisten = self.contentHistory.listen(setContext);
      }
    },
  };

  return <SetRouter {...props} lifecycle={mergelifecycle} />;
};

export default SetRouterWrap;
```
