## 1. 模块定位与入口


**事件中心**用于搭建平台内的 **事件驱动** 能力：将「**谁在什么时机发生了什么**」与「**执行哪段逻辑**」解耦——先注册可复用的 **事件执行程序**，再通过 **事件订阅** 绑定到具体 **表单** 或 **流程** 的 **触发时机**。

**典型入口**：


- 顶栏主菜单进入 **「事件中心」**（与 **首页**、**应用** 等并列，以环境为准）。
- 应用能力集内也可能挂载同一套页面，路由常含 **`eventCenter`** 段（见附录）。

## 2. 左侧菜单结构（总览）


以下为截图中可见的侧边栏分组（名称以线上为准）：


| 分组/项 | 职责概要 |
| --- | --- |
| **事件管理**（可折叠） |  |
| └ **事件执行程序** | 定义「被调用的逻辑单元」：名称、编码、类型、绑定服务/资源、入参 |
| └ **事件订阅** | 定义「何时触发」→ 关联执行程序；支持检索、批量启停 |
| └ **执行日志** | 查看订阅触发与执行结果历史 |
| └ **业务编排** | 复杂编排能力（与事件配合，以产品文档为准） |
| └ **业务编排日志** | 编排执行历史 |
| **连接器工厂** | HTTP/DATABASE 等连接器（见独立文档 `connector-factory-features.md`） |
| **触发位置管理** | 管理可挂载触发点的业务位置/场景 |
| **连接管理** | 如 **连接产品** 子项 |
| **数据源管理** / **数据管理** | 数据源相关（截图表述差异以线上为准） |

## 3. 核心概念：执行程序 vs 订阅


| 术语 | 是什么 | 类比 |
| --- | --- | --- |
| **事件执行程序** | 平台可调用的 **函数/服务绑定**（内置程序、参数契约等） | 「可被调用的子程序」 |
| **事件订阅** | **订阅名称 + 适用范围 + 资源（表单/流程）+ 触发时机 [+ 指定节点] + 一个或多个执行程序** | 「当…则调用…」的规则 |


**配置顺序建议**：先在 **事件执行程序** 中登记程序 → 再在 **事件订阅** 中 **新增** 并勾选/添加对应 **执行程序**。

## 4. 功能速查（关键词 → 路径 → 是否支持）


| 用户可能问的关键词 | 路径 | 是否支持（据截图） |
| --- | --- | --- |
| 查现有订阅 | **事件订阅** → 筛选区 + **查询** | **支持** |
| 新建订阅 | **新增** | **支持** |
| 批量启用/禁用 | 勾选行 → **启用** / **禁用** | **支持** |
| 编辑/删除单条 | 表格 **操作** 图标 | **支持** |
| 表单保存前插逻辑 | 订阅 → **表单事件** → **创建前事件** 等 | **支持**（选项以线上为准） |
| 流程审批后插逻辑 | **流程事件** → **审批后** + 可选 **指定节点触发** | **支持** |
| 登记可复用逻辑 | **事件执行程序** → **新增** / 编辑 | **支持** |
| 内置程序绑服务 | 执行程序 → **程序类型** **内置程序**、**所属服务**、**绑定资源** | **支持** |
| 看执行历史 | **执行日志** | **支持** |

## 7. 编辑订阅：流程事件


当 **适用范围 = 流程事件**（或 **流程事件**）时：


| 字段 | 必填 | 说明 |
| --- | --- | --- |
| **名称** | * | 同上 |
| **适用范围** | * | **流程事件** |
| **选择流程** | * | 绑定流程模板（如「教师流程」） |
| **触发时机** | * | 可**搜索**的下拉；挂在流程/节点生命周期的具体时点 |


### 7.1 触发时机（流程类，截图归纳）


>


下列按**语义分组**整理，便于 AI 与用户理解；**完整且唯一**的清单以线上下拉为准（滚动区可能还有更多项）。


#### （1）节点级（常与「指定节点触发」配合）


| 选项（文案） | 含义概要 |
| --- | --- |
| **节点进入** | 流程流入某一节点时 |
| **节点退出** | 流程离开某一节点时（如提交/通过后离开） |
| **流程节点更新** | 节点内信息或状态发生更新时 |


#### （2）流程实例级（偏整单状态）


| 选项（文案） | 含义概要 |
| --- | --- |
| **废弃前** / **废弃后** | 流程实例被**废弃**前一刻 / 完成之后 |
| **结束后** | 流程到达**最终结束**状态之后 |
| **流程重启前** / **流程重启后** | 对已停/结流程做**重启**前 / 重启完成之后 |


#### （3）审批与草稿、驳回、转办


| 选项（文案） | 含义概要 |
| --- | --- |
| **审批前** / **审批后** | 某审批动作发生前 / 完成之后（**审批后**较常用） |
| **审批撤回前** / **审批撤回后** | **审批撤回**动效前 / 之后 |
| **草稿提交前** / **草稿提交后** | **草稿提交**前一刻 / 提交完成之后 |
| **驳回前** / **驳回后** | **驳回**发生前 / 完成之后 |
| **转办后** | **转办**完成之后 |


#### （4）起草人撤回（发起人）


| 选项（文案） | 含义概要 |
| --- | --- |
| **起草人撤回前** / **起草人撤回后** | **起草人**将实例撤回前 / 撤回完成之后 |


**说明**：同一环境下选项可能随版本增减；配置时可用下拉内 **搜索框** 过滤关键字（如「驳回」「审批」）。

### 7.2 指定节点触发


- 字段 **「指定节点触发」**：将触发限定在某一 **流程节点**，而非全流程任意时刻。
- **与 7.1 的关系**：**节点进入 / 节点退出 / 流程节点更新** 等节点级时机，通常需结合本字段才能精确到「哪一个节点」；**审批后、结束后** 等既可全天梯也可配合节点缩小范围（以产品实际行为为准）。
- 截图示例：**审批节点 (n_sys_suspend)**、**结束 (n_sys_end)**、**开始 (n_sys_start)**、**起草节点 (n_sys_draft)** 等（带节点 **技术 ID**）。


### 7.3 执行程序


- 下拉/列表选择一条或多条执行程序（示例项：**语言资源的新增和更新**、**天舟云事件触发**、导入/导出测试事件、仓库渲染事件等）。
