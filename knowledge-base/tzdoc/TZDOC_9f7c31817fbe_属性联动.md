## 1. 模块概述


**属性联动**用于配置表单字段之间的动态关系：当某一属性在指定 **事件** 下发生交互或表单处于某种生命周期状态时，在满足 **附加条件** 的前提下，对 **影响属性** 执行一类或多类 **联动动作**（改值、改可选值、改控件属性、请求、脚本等）。


**典型价值**：


- 根据 A 字段取值，显示/隐藏、只读/可编辑、必填 B 字段（通过「属性改变」等动作实现，具体以动作类型选项为准）。
- 级联下拉：父字段变化后刷新子字段可选值（「可选值改变」等）。
- 跨字段赋值、调用接口取数回填（「查询并赋值」「发起请求」等）。
- 复杂规则用 **JavaScript 自定义** 脚本实现，并可借助 **AI 编程助手** 生成片段。

## 2. 入口与列表页


### 2.1 如何进入


在表单相关设计上下文中，顶部导航选中 **「属性联动」** Tab（与 **表单设计**、**数据列表**、**流程设计**、**表单转换** 等并列；当前以该 Tab 为配置入口）。


### 2.2 列表页工具栏


| 按钮 | 说明 |
| --- | --- |
| **新增联动** | 新建一条联动规则，进入或打开「联动详情」进行配置 |
| **刷新** | 刷新联动规则列表 |
| **AI 协助** | 平台提供的 AI 辅助能力（具体能力以产品说明为准） |


### 2.3 列表表格列（规则摘要）


列表用于总览已配置的规则，列含义如下（便于用户描述「我想改备注/看谁改的」）：


| 列名 | 含义 |
| --- | --- |
| **操作** | 单条规则的维护入口（如编辑、删除等，以界面按钮为准） |
| **触发属性** | 作为触发入口的表单属性/字段 |
| **事件** | 触发时机（与详情页「触发动作」对应，列表汇总展示） |
| **条件** | 附加条件摘要 |
| **影响属性** | 被联动影响的属性摘要 |
| **启用状态** | 规则是否生效（可禁用/启用类控制） |
| **备注** | 规则说明 |
| **操作人** | 最近维护人 |
| **操作时间** | 最近维护时间 |


**空列表**：无数据时表格为空，通过 **新增联动** 开始配置。

## 4. 联动详情：整体结构


**联动详情**为配置单条联动规则的主界面（弹窗/抽屉形态，标题区常见 **「联动详情」**，可关闭）。


### 4.1 顶栏按钮（从右至左常见顺序）


| 控件 | 作用 |
| --- | --- |
| **保存** | 提交当前联动配置；部分环境带下拉子项（如更多保存方式） |
| **收起所有组** | 折叠全部 **联动组**，便于浏览多条组 |
| **新增联动组** | 在同一联动规则下新增一组「附加条件 + 联动属性」 |
| **禁用** | 禁用当前联动规则（与列表「启用状态」呼应） |
| **返回** | 返回列表或上一级 |


### 4.2 三大配置层次（给 AI 的固定话术）


- **全局**：**触发属性** + **触发动作** + **简介** —— 决定「在什么时机尝试执行本联动」。
- **联动组（可多个）**：每组有 **描述**，组内包含 **附加条件** 与 **联动属性**。
- **组内逻辑**：**附加条件** 为 IF；**联动属性** 为 THEN；多行条件、多行动作一般可按行 **排序 / 复制 / 删除**。

## 5. 全局触发设置


### 5.1 字段说明


| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| **触发属性** | 必填（标 *） | 选择作为触发入口的表单属性/组件 |
| **触发动作** | 必填（标 *，旁有帮助图标） | 指定在何种交互或表单生命周期事件上触发 |
| **简介** | 选填 | 对整条联动规则的简短说明 |


### 5.2 触发动作（可选值汇总）


下列选项来⾃截图中 **触发动作** 下拉面板的可见项（产品若增删选项，以线上为准）：


| 选项（文案） | 含义概要 |
| --- | --- |
| 属性组件失去点击焦点 | 失焦（blur）时触发 |
| 属性组件获得点击焦点 | 聚焦（focus）时触发 |
| 属性组件值被改变时 | 值变更时触发 |
| 属性组件触发搜索时 | 搜索型控件触发检索时 |
| 属性所在表单初创状态被打开时 | 表单**新建**打开时 |
| 属性所在表单编辑状态被打开时 | 表单**编辑**打开时 |
| 属性所在表单被关闭时 | 表单关闭时 |
| 属性所在表单初创状态被关闭时 | 表单在**新建**态关闭时 |

## 6. 联动组


每条联动可包含 **一个或多个联动组**（界面标题如 **「联动组1」**）。


### 6.1 组头操作


组标题旁常见图标能力（以界面为准）：


- **新增**：在组维度快速添加（若与顶栏「新增联动组」并存，顶栏为新增整个组，组内「+」多为条件/动作行级新增）。
- **复制**：复制当前组配置。
- **删除**：删除该组。
- **收起/展开**：折叠组内 **附加条件** 与 **联动属性**。


### 6.2 描述


**描述**：大块文本区，用于记录该组业务意图、边界说明，便于后续维护与 AI 理解上下文。

## 7. 附加条件（IF）


**附加条件**定义：在全局「触发属性 / 触发动作」发生后，**还需要满足**哪些条件才执行本组内的联动属性。


### 7.1 行级操作


每行条件左侧通常支持：


- **排序**（上下移动，影响组合求值顺序或展示顺序，以引擎规则为准）。
- **复制**
- **删除**


### 7.2 行内字段


| 字段 | 说明 |
| --- | --- |
| **条件属性** | 参与判断的表单属性 |
| **运算规则** | 比较/包含等运算符 |
| **动作类型** | 条件行的处理/作用域类型（截图示意中有 **「表单」** 类取值，具体枚举以线上为准） |
| **属性赋值**（部分场景出现） | 与条件比较的目标值、表达式或赋值来源（以配置向导为准） |


### 7.3 运算规则（截图中可见枚举）


下列运算规则在截图 **附加条件 → 运算规则** 中出现（若产品扩展，以线上为准）：


- 等于
- 不等于
- 包含
- 不包含
- 大于
- 大于等于
- 小于
- 小于等于


### 7.4 多条件组合


通过行首 **「+」** 增加多行条件。多条条件之间如何 AND/OR 组合，若在界面上有「且/或」总开关请在实际产品中确认；**若未在界面展示，应答时应提示用户以产品规则或实施文档为准。**

## 8. 联动属性（THEN）


**联动属性**定义：当**全局触发成立**且 **附加条件满足** 时，对哪些目标属性执行何种动作。


### 8.1 行级操作


与附加条件类似：支持 **拖拽/排序、复制、删除**（以图标为准）。


### 8.2 动作（联动类型）枚举


**动作** 下拉常见选项（多图合并；以线上完整列表为准）：


| 动作 | 典型用途 |
| --- | --- |
| **值改变** | 修改目标字段取值 |
| **可选值改变** | 修改下拉/枚举等 **可选数据集**（级联选项等） |
| **属性改变** | 修改字段 **元数据/展示属性**（如只读、隐藏、必填等，具体子类型由「动作类型」再细分） |
| **查询并赋值** | 根据查询结果 **给字段赋值** |
| **发起请求** | 触发服务端或集成请求 |
| **自定义** | 使用 **JavaScript** 编写复杂逻辑（见第 9 节） |


### 8.3 行内其它字段


| 字段 | 说明 |
| --- | --- |
| **目标属性** | 被影响的字段 |
| **动作类型** | 对「值改变 / 可选值改变 / 属性改变」等的进一步细分（如「赋值」等） |
| **属性赋值** | 在「可选值改变」等场景下，用于填写赋值表达式或常量（长文本框） |


### 8.4 示例（来自截图描述，仅作理解）


- 一行 **动作 = 值改变**，指定 **目标属性**，再选 **动作类型**。
- 一行 **动作 = 可选值改变**，**目标属性** 如「分类」，**动作类型** 如「赋值」，并在 **属性赋值** 中写入具体配置。

## 9. 自定义动作与脚本编辑器


当 **动作** 选择 **自定义** 时，界面展开 **代码编辑器**（深色主题示意）。


### 9.1 编辑器能力


| 能力 | 说明 |
| --- | --- |
| 语言 | **JavaScript** |
| **全屏** | 放大编辑区域 |
| **版本历史** | 查看与恢复历史脚本版本 |
| **AI 助手** / **AI 编程助手** | 辅助生成/改写脚本；可能以浮层面板展示 |


### 9.2 AI 编程助手快捷片段（截图中出现的示例）


快捷入口（名称以界面为准）包括但不限于：


- 弹窗打开详情
- 弹窗打开列表
- 弹窗打开视图
- 发起请求
- **更多**（扩展模板）


**应答提示**：若用户问「能不能用 AI 写脚本」，可答：**支持在自定义动作中打开 AI 助手**；具体生成质量、权限与合规要求以公司规范为准。

## 10. 逻辑模型小结（给 AI）


用一句话描述执行模型，便于问答一致性：


>


**当【触发属性】发生【触发动作】时，对每一个【联动组】：若【附加条件】全部（或按产品规则组合）满足，则依次执行【联动属性】中配置的各【动作】对【目标属性】的影响；若某动作为【自定义】，则执行对应 JavaScript。**


列表页字段与详情对应关系记忆：


- 列表 **「事件」** ≈ 详情 **「触发动作」**
- 列表 **「条件」** ≈ 附加条件摘要
- 列表 **「影响属性」** ≈ 联动属性摘要
