# 导入报错常见问题

# 天舟云列表导入 — 报错排查指引


>


列表数据从 Excel 导入时可能出现报错，本文档汇总常见报错场景与排查步骤，帮助用户自助解决问题。


---


## 一、弹窗报错：`Cell index must be >= 0`


### 现象


导入时直接弹出错误框，提示 **`Cell index must be >= 0`**。


### 原因


Excel 中存在格式异常的单元格（如隐藏损坏区域、异常合并等），导致解析时行列索引非法。


### 解决方法


- 选中 Excel 中 **实际需要导入的数据区域**（表头 + 数据行）
- **复制**并粘贴到一个 **新建的空白 Sheet** 中
- **删除原有的问题 Sheet**，仅保留新 Sheet
- 保存后**重新导入**


---


## 二、导入失败：`属性:[xxx]:值[yyyyy]不正确`


### 现象


导入失败，下载失败结果 Excel 后，报错信息为：


>


**属性:[xxx]:值[yyyyy]不正确**


其中 `xxx` 为字段名，`yyyyy` 为导入时该字段填写的值。


### 排查步骤


按以下顺序逐步排查：


#### 步骤 1：确认值能否在页面上搜索到


- 打开对应业务的 **创建页面**
- 在字段 `xxx` 的搜索框中输入 `yyyyy`，观察能否正常搜出


| 搜索结果 | 下一步 |
| --- | --- |
| **搜不到** | ① 检查 `yyyyy` 是否为正确的值；② 检查该字段的**数据集配置**是否正确 |
| **能搜到** | 继续步骤 2 |


#### 步骤 2：检查字段是否存在依赖查询


部分字段的数据集查询**依赖其他字段的值**（如先选组织再选人员）。**导入时不支持这种带参的级联查询**，会导致校验失败。


>


**示例：** 字段「人员」的可选值依赖字段「组织」的已填值，导入时无法模拟这种交互，因此报错。


若字段无依赖关系，继续步骤 3。


#### 步骤 3：按字段类型检查是否支持 name → code 反查


导入时系统需要将 Excel 中的**展示名称（name）** 转换为内部**编码（code）**。如果底层不支持按 name 查 code，就会报「值不正确」。


根据字段类型分别检查：


| 字段类型 | 检查要点 |
| --- | --- |
| **单选搜索 / 多选搜索** | 检查数据字典接口是否实现了**从 name 查 code**（参考：[数据字典接口规范](https://kb.cvte.com/pages/viewpage.action?pageId=229200383)） |
| **API 查询** | 检查接口是否支持通过 **name 查询 code** |
| **连接器** | 检查连接器的 SQL 是否支持 **name → code** 的反查 |
