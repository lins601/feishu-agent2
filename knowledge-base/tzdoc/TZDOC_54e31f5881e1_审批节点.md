# 审批节点


### 功能介绍


审批人的职责是处理该节点上的审批任务，需要做出同意、驳回、转办等决策，你可以根据需要自行增删审批节点


### 使用场景


流程流转到该节点时，会停留在该节点，等待审批人的决策。


### 操作步骤


##### 一、添加审批人节点


##### 二、设置审批人


在不同的业务场景中，审批人所对应的角色也不尽相同，天舟云审批人节点目前支持配置以下审批人类型


1、指定人员


```
该流程节点的审批人为固定审批人，支持多选
```


2、指定角色


```
指定角色审批，将多个人进行标记，可以使用CSB角色
```


3、流程创建人


```
当前节点的审批人为流程发起者。可用于需要发起人重新进行信息复核的场景
```


4、表单字段


```
 把表单内人员字段作为审批人。
```


5、连接器


```
 支持从自定义连接器中获取审批人
```


##### 三、设置审批按钮(非必需)


审批按钮是当流程运行到当前节点时，节点审批人可以执行的审批操作，目前已支持通过、驳回、转办、转交等操作，你可以配置某个操作按钮的显示名称、默认审批意见、是否启用等


##### 四、设置字段权限


流程节点的字段权限控制可以控制审批流程中某个审批节点可以查看的表单字段权限，支持根据组件名称配置可见、可编辑和必填三种状态。


##### 五、设置自动审批(非必需)


 流程节点支持配置当前节点的自动审批规则、处理人身份重复时是否跳过逻辑
