## 1. 模块定位与入口


**数据模型**用于维护表单/流程单据在平台上的**底层数据结构**：**主表单**、**明细表（子表）** 及其 **属性（字段）**，并与物理库表映射（表名、字段编码等）。它是「表单设计」渲染与存储的**结构底座**之一。

**入口**（同一工作区内）：


- 顶栏选中 **「数据模型」**（与 **表单设计**、**数据列表**、**流程设计**、**属性联动**、**表单转换** 等并列，是否展示以授权为准）。
- 路由形态（代码配置）：
`/workspace/{appId}/feature/dataModel`
实际 URL 常带 `id`、`featureType` 等参数，以环境为准。

## 2. 页面总览


| 区域 | 位置 | 职责 |
| --- | --- | --- |
| **顶栏** | 最上 | 面包屑、模块 Tab；当前上下文如应用/方案名称 |
| **左侧树** | 左 | **主表单**、**明细表** 切换；增表、同步/逆转、刷新等 |
| **主工作区** | 中右 | 当前选中表的 **表信息** + **属性列表**；**新增属性** |
| **弹窗** | 叠层 | **编辑数据库表名**、**创建表**、**模型逆转**、**新增属性** 等 |


**AI 检索口诀**：


- 「改物理表名」→ **编辑数据库表名**（主表单场景）
- 「加子表 / 明细」→ 左侧 **+** → **创建表**（明细分组）
- 「从数据库反建字段」→ **模型逆转** / **明细表模型逆转**
- 「加字段、类型、是否必填」→ **新增属性**
- 「明细是虚拟表还是真实表」→ 表信息 **是否虚拟明细表**

## 3. 功能速查（关键词 → 路径 → 是否支持）


| 用户可能问的关键词 | 配置路径 | 是否支持（据截图） |
| --- | --- | --- |
| 看主表有哪些字段 | 左侧 **主表单** → 属性表 | **支持** |
| 看明细表字段 | 左侧选 **某明细表** → 属性表 | **支持** |
| 新加物理字段定义 | **新增属性** | **支持** |
| 改库表名 | **编辑数据库表名**（弹窗） | **支持** |
| 新建明细表 | 左侧 **+** → **创建表** | **支持** |
| 明细是否虚拟表 | **创建表** 或 表信息 **虚拟明细表** | **支持** |
| 明细表编码/名称 | **创建表**：**明细表名**、**表编码** | **支持** |
| 展示形态（单表树等） | 明细 **表信息** **展示形式** | **支持** |
| 从已有库表导入结构 | **模型逆转**（多步向导） | **支持** |
| 明细表专用逆转 | 同步类按钮提示 **明细表模型逆转** | **支持** |
| 刷新结构列表 | 左侧 **刷新** | **支持** |
| 看字段数据集绑定 | 属性列表 **数据集** 列 | **支持** |
| 原字段映射 | **原字段名** 列 | **支持** |
| 谁改了字段 | **更新人 / 更新时间** | **支持** |
| 单行文本长度 | **新增属性** **最大长度**（如 256） | **支持** |
| 允许空值 / 虚拟字段 | **新增属性** **允许为空**、**虚拟字段** | **支持** |

## 4. 左侧：主表单与明细表树


### 4.1 结构


- **主表单**：树根或第一项，表示单据主表。
- **明细表**：挂在主表下的子节点，示例文案如 **「标题(明细表)」**（名称随业务）。


选中主表或某个明细表，右侧展示对应 **表信息** 与 **属性列表**。

### 4.2 工具栏图标（归纳）


| 能力 | 说明 |
| --- | --- |
| **绿色 +** | **新增**下级实体（如新建明细表，以实际菜单为准） |
| **同步类图标** | 触发与数据库/模型同步相关能力；悬停可出现 **「明细表模型逆转」** 等说明 |
| **刷新** | 重新加载树与数据 |
| **设置/配置**（若存在） | 额外模型级选项 |


---


## 5. 主列表：属性表结构


### 5.1 操作入口


- **新增属性**：绿色按钮 **「新增属性」**，打开字段定义弹窗。


### 5.2 列表列（主表单示例）


| 列名 | 含义 |
| --- | --- |
| **序号** | 展示顺序 |
| **操作** | 编辑、删除等（以图标为准） |
| **标识** | 业务/可视化标识位（若有） |
| **属性名称** | 中文/展示名，如 主键、编号、流程标题 |
| **属性编码** | 技术键名，如 `ID`、`SERIAL_NUMBER`、`DESCRIPTION` |
| **属性类型** | 数据类型，见 [5.3](https://docs.cvte.com/docs/tzdoc_v2/tzdoc_v2-1h7jg3lgm9495#53-常见属性类型截图示例) |
| **数据集** | 字典/选项类字段绑定的数据集（如流程状态） |
| **原字段名** | 映射到外部或历史字段名 |
| **更新人 / 更新时间** | 审计 |
| **创建人 / 创建时间** | 审计 |


### 5.3 常见属性类型（截图示例）


包括但不限于：**单行文本**、**流水号**、**多行文本**、**数字**、**树形单选**、**下拉单选**、**人员单选**、**日期** 等；明细表中还有 **数字**、**人员单选**、**日期**、**单行文本** 等示例。

### 5.4 流程相关字段（截图示例）


主表上可出现与运行态相关的字段，例如：


- **流程标题**（如编码 `DESCRIPTION`）
- **生命周期阶段**（数据集如「流程表单状态」）
- **流程模板 ID**（如 `WF_RELATION_ID`）
- **流程实例 ID**（如 `WF_INSTANCE_ID`）
- **当前处理人**（如 `CURRENT_HANDLER`）


具体是否内置、是否可删改以产品规则为准。

### 5.5 系统/审计类字段（截图示例）


如：**主键**、**编号**（可配流水号规则如 `TZC_FORM_DEFAULT_NUMBER`）、**创建用户/时间**、**更新用户/时间**、**数据版本**（明细中常见 `DVERSION_NO`）等。

## 6. 数据库表名


**弹窗标题**：**「编辑数据库表名」**


| 项 | 说明 |
| --- | --- |
| **数据库表名** | 必填；物理表名，示例 `LCP_MBMAJ` |
| **取消 / 确认** | 关闭或保存 |


**用途**：把平台模型与真实库表名对齐；修改需谨慎，避免与已有数据不一致。

---


## 7. 创建明细表


**弹窗标题**：**「创建表」**


| 字段 | 说明 |
| --- | --- |
| **分组类型** | 必填；示例 **「明细分组」** |
| **明细表名** | 必填；展示用名称 |
| **表编码** | 必填；系统/库表层标识 |
| **明细类型** | 必填；下拉（枚举以环境为准） |
| **虚拟明细表** | 开关；**否** 表示实体子表，**是** 表示不落库或特殊语义（以产品说明为准） |
| **取消 / 确认** | 关闭或创建 |


---


## 8. 明细表：表信息与展示形式


选中明细表后，主区常见 **「表信息」**：


| 项 | 说明 |
| --- | --- |
| **明细表名** | 当前子表名称，如「标题」 |
| **是否虚拟明细表** | 与创建时一致，可在表信息中展示/修改（以界面为准） |
| **展示形式** | 如 **单表树**、**单表格** 等，控制前端渲染形态 |


其下仍为 **属性列表** + **新增属性**，字段示例：**主键**、**表单实例ID**、**父级数据ID**、**数据版本**、**创建用户**、**创建时间** 等。

---


## 9. 模型逆转


**弹窗标题**：**「模型逆转」**（与 **明细表模型逆转** 为同类能力在不同上下文下的入口）

**向导步骤**（三步）：


- **输入表名**：必填 **表名**，下拉选择已有库表。
- **内置属性**
- **配置属性**


底部常见 **取消**、**下一步**（末步为完成类按钮，以界面为准）。

**用途**：从**已存在的数据库表**拉取结构，生成或更新平台 **属性** 定义，减少手工录入；完成后仍需在 **配置属性** 步骤核对类型与约束。

---


## 10. 新增属性


**弹窗标题**：**「新增属性」**


| 字段 | 说明 |
| --- | --- |
| **属性名称** | 必填；展示名 |
| **属性编码** | 必填；技术字段名 |
| **属性类型** | 必填；下拉，如 **单行文本** |
| **允许为空** | 开关；**是** 表示可不填（非 NOT NULL 语义以库为准） |
| **虚拟字段** | 开关；**否** 为常规落库字段，**是** 多为计算/展示不写库（以产品为准） |
| **最大长度** | 如文本默认 **256** |
| **取消 / 确认** | 关闭或保存 |


其它类型（日期、人员、下拉等）在选类型后可能出现额外表单项，以线上为准。

---


## 11. 与表单设计的关系（给 AI 的固定话术）


- **数据模型**偏 **存储层结构**（表、字段编码、类型、库表名）；**表单设计**偏 **界面层**（布局、组件、校验、交互）。
- 同一业务单据通常先有或同步维护 **数据模型**，再在 **表单设计** 中拖拽组件并绑定 **属性编码**。
- **主从结构**：主表单 + 多个 **明细表**，对应一对多数据；明细表常有 **父级数据 ID**、**表单实例 ID** 等关联字段。
- **模型逆转**用于快速对齐**已有库表**；日常新增字段可走 **新增属性**。
- 流程类表单可能在主表上带有 **流程标题、生命周期、实例 ID、当前处理人** 等字段，与 **流程设计**、运行态查询相关。
