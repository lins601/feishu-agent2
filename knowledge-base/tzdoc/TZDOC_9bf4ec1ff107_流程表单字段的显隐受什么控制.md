# 流程表单字段的显隐受什么控制

#### 目标


本文档用于说明流程表单中的字段显隐、是否只读、是否必填和什么东西相关


#### 总体逻辑


- 表单设计中字段配置的字段权限: 隐藏、显示、可编辑、必填
- 高级管理->权限设置->配置的字段权限:隐藏、显示、可编辑、必填
- 流程节点中配置的字段 隐藏、显示、可编辑、必填


  - 仅对以下流程 **参与者** 生效


    - **流程当前处理人**：仅以当前节点配置的字段权限为准
    - **流程历史处理人**：包括创建人,用户处理过的节点的权限设置，所有权限高于只读会降为只读，如 可编辑 会降为 “只读”


- 前端配置的属性联动


**备注说明**:


- **流程当前处理人** 以 3 为准
- **流程历史处理人**会取2和3中的权限并集往上取大的字段权限
- **非流程参与者**字段权限以2的设置为准
- 如果在4中有配置字段的 隐藏、显示、可编辑、必填，则以4为准。


**举个排查例子**:


```
排查流程结束用户A看到某字段为啥可编辑?
思路:
1.流程结束会走到结束节点，结束节点不能设置节点权限，所以此时用户的身份不是 当前处理人，
2.可能是 流程历史处理人和非流程参与者
  -  流程历史处理人字段权限最大为只读，该问题为可编辑，排除 流程历史处理人身份影响，剩下非流程参与者
3.非流程参与者怎么检查?
  - 检查高级管理->权限设置->非创建态字段权限，
  - 检查属性联动是否有对该字段控制
```
