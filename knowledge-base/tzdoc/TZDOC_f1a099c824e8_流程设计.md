## 1. 模块定位与入口


## **流程设计**用于在流程类表单（工作流表单）上，**可视化编排**审批/办理过程：定义节点、转移（连线标签如「提交」「通过」）、**节点经办人**、**字段权限**、**操作按钮**、**消息**等，并与**表单设计**中的单据字段联动。


## 2. 页面总览（布局与心智模型）


| 区域 | 位置 | 职责 |
| --- | --- | --- |
| **顶栏** | 最上 | 面包屑、模块 Tab、**额外属性配置**、**当前版本**与状态、**版本描述**、**保存** |
| **画布工具条** | 画布上方/左上 | 撤销/重做、复制/粘贴/删除、缩放、适应屏幕、清空、新增节点类型（开始/任务/网关/子流程等，以图标为准） |
| **流程画布** | 中部 | 网格背景上的**节点**与**连线（边）**；连线上可有**动作标签**（如「提交」「通过」） |
| **提示条** | 画布上方 | **已启用**流程时提示须**新建版本**才能增删节点/连线 |
| **右侧配置面板** | 右侧 | 选中节点后：**基础配置 / 字段权限 / 操作按钮 / 自定义消息** 等 Tab |


**AI 检索口诀**：


- 「改审批人、并行还是会签」→ 选中节点 → **基础配置**
- 「这一步字段能不能改」→ **字段权限**
- 「这一步有哪些按钮、谁可操作」→ **操作按钮**
- 「待办/审批标题文案」→ **自定义消息** 或 **额外属性配置** 下的待办模版
- 「流程上线了不能改结构」→ **版本** → **新建版本**
- 「多套流程、查列表」→ **管理多工作流**

## 4. 顶栏：版本、保存与全局配置


### 4.1 常用控件


| 控件 | 说明 |
| --- | --- |
| **额外属性配置** | 下拉菜单，见 [4.2](https://docs.cvte.com/docs/tzdoc_v2/tzdoc_v2-1h7je7h0d9mdr#42-额外属性配置下拉项) |
| **版本 Vn（启用中）** | 当前编辑/启用版本；点击常进入版本管理或切换 |
| **版本描述** | 维护本版本说明 |
| **保存** | 持久化当前设计（具体校验以产品为准） |


### 4.2 「额外属性配置」下拉项


| 菜单项 | 用途概要 |
| --- | --- |
| **驳回原因配置** | 维护驳回时可选项/标准化原因，便于统计与体验统一 |
| **业务回写属性配置** | 流程节点与业务对象之间的**回写**映射 |
| **额外配置** | 其它流程级参数入口 |
| **启用流程激活** | 开关；控制是否需要/允许「流程激活」类机制（语义以产品说明为准） |
| **自定义待办模版** | 配置待办列表展示的标题、摘要等模版，提升可读性 |

## 5. 画布与流程建模


### 5.1 启用中流程的编辑约束


画布上方常见 **黄色提示条**，大意：


>


**流程已启用**，若需**增删节点和连线**，请先**添加新版本**。


**给 AI 的话术**：已启用版本通常**锁定拓扑**；改结构需 **新建版本** 后再编辑。


### 5.2 典型节点与转移（截图示例）


自上而下示例（真实名称以业务配置为准）：


| 元素 | 类型说明 |
| --- | --- |
| **开始** | 起始椭圆 |
| **启动事件** | 流程启动相关事件节点 |
| **起草节点** | 拟稿/填报任务节点 |
| **提交** | 连线上动作标签，起草 → 下一节点 |
| **审批节点** | 审批类任务节点（可选中高亮） |
| **通过** | 连线上动作标签，审批 → 结束 |
| **结束** | 结束椭圆 |


节点可选用不同图标（如公文包表示审批）；选中节点可调出右侧配置，部分模式下节点旁有删除等操作（新建版本/可编辑状态下）。


### 5.3 画布工具能力（归纳）


- **历史操作**：撤销、重做
- **编辑**：复制、粘贴、删除选中元素
- **视图**：放大、缩小、适应屏幕、平移（若有手型工具）
- **建模**：从工具条插入开始/任务/网关/子流程等（以图标提示为准）

## 6. 右侧节点配置面板


选中画布上某一节点后，右侧展示该节点属性。常见 **主 Tab**：


### 6.1 基础配置


| 配置项 | 说明 |
| --- | --- |
| **节点名称** | 展示用名称 |
| **节点 id** | 系统侧唯一标识（如 `n_sys_suspend`） |
| **节点经办人** | 常 **必填**；可对接人员、角色、**连接器**等（截图示例为连接器选项） |
| **处理人为空时自动跳过** | 无经办人时是否跳过该节点 |
| **审批类型** | **并行** / **会审（串签）** 等（截图中为二选一） |
| **重复处理人控制** | 如 **处理人重复不跳过** 等策略，下拉选择 |
| **加载更多高级配置** | 展开更多节点级高级项 |


### 6.2 字段权限


- **子 Tab**：**字段** | **按钮**
- **范围**：下拉选择绑定的单据/流程（如当前流程模板名）
- **搜索**：快速定位表单组件
- **表格列**（字段子 Tab）：**组件名称**、**可见**、**可编辑**、**必填**（勾选）；支持表头 **全选** 行批量勾选


**用途**：控制**当前节点处理人**在表单上看到什么、能否改、是否必填。


### 6.3 操作按钮


每个操作一般为**独立区块**，顶部有**总开关**。常见操作（以实际启用为准）：


| 操作 | 配置要点 |
| --- | --- |
| **废弃** | 显示名称、审批意见/必填/默认文本、**可操作角色**（如创建人、处理人） |
| **驳回** | 同上；可选 **自定义驳回节点**、**驳回重新提交时机**（如按顺序执行） |
| **撤回** | **可操作角色** 如 **创建人**、**已处理人** |
| **转办** | **处理人** 等；**自定义用户列表** 限制转办候选人 |
| **通过** | 通常 **处理人** 可点；意见区可选 |
| **催办** | 示例中 **创建人** 可催办；意见可关 |


**子项通用模式**：


- **显示名称**、**默认文本** 支持 **多语言**（翻译图标）
- **审批意见**：是否展示意见框；**审批意见必填**；**默认文本**（如「同意」）


### 6.4 自定义消息


- **列表区**：**添加** 多条消息规则
- **字段示例**：**类型**（如 **审批**）、**消息头**、**消息体**
- **弹窗「自定义消息」**：大文本编辑区 + **可用变量**


  - **表单变量** / **流程变量**（如 **提交人**、**当前节点名称**、**流程标题**、**流程模板名称**、**操作处理人** 等）
  - 支持 **搜索变量**；变量可标记 **自定义** 映射/格式


**用途**：配置节点相关通知/待办的动态文案。

## 7. 版本管理弹窗


**标题**：**管理流程版本**


| 能力 | 说明 |
| --- | --- |
| **+ 新建版本** | 创建新版本以继续改节点/连线或迭代配置 |
| **导出** | 导出流程定义（用于备份/迁移，格式以产品为准） |
| **列表列** | **版本序号**（如 流程版本V1/V2）、**状态**、**操作** |
| **状态示例** | **老版本**、**启用中**；可同时标注 **当前编辑版本** |
| **操作** | **启用**、**编辑**、**删除**（启用中或当前编辑版本上 **删除** 常为禁用） |
| **取消** | 关闭弹窗 |

## 8. 管理多工作流


**标题**：**管理多工作流**


用于在同一应用/单据维度下维护**多条流程模板**（与单个流程画布配合：先列表再进设计器）。


| 区块 | 说明 |
| --- | --- |
| **查询条件** | **流程名称**、**状态**、**使用单据**（下拉）；**查询** / **重置**；可 **展开** 更多筛选 |
| **新建** | 创建新工作流 |
| **表格列** | **序号**、**操作**（编辑/删除）、**流程名称**（可点进设计）、**流程编码**、**状态**（如启用）、**流程描述**、**使用单据及条件**（如某单据 **无条件**）、**创建时间**、**更新时间** |
| **分页** | 总数与每页条数 |

## 9. 逻辑与协作关系（给 AI 的固定话术）


- **流程设计**依赖 **表单设计** 提供的**单据与字段**；**字段权限**、**操作按钮**中的「字段/按钮」与子页签与表单组件树对应。
- **启用中**版本侧重稳定运行；**拓扑变更**走 **新版本**。
- **单节点** = **基础配置**（谁办、怎么办） + **字段权限**（数据规则） + **操作按钮**（出口动作） + **消息**（通知体验）。
- **额外属性配置**偏**流程级**横切能力（驳回原因、回写、待办模版等）。
- **管理多工作流**偏**流程模板集合**维度的检索与生命周期，与 **管理流程版本**（单流程多版本）是不同粒度。
