## 1.1 请求数据说明 `TodoReq`


| 参数名 | 数据类型 | 是否必填 | 说明 | 备注 |
| --- | --- | --- | --- | --- |
| sid | String | 是 | 系统单据号 | 不能为空，最多64个字符 |
| title | String | 是 | 标题 | 不能为空，最多1024个字符 |
| todoUrl | String | 是 | 待办链接 | 不能为空，最多1024个字符 |
| todoType | String | 是 | 待办类型 | 可选值：`processApproval`（流程审批）/ `dealTask`（任务处理）/ `doneManual`（可手动完结） |
| todoLv | String | 是 | 待办等级 | 不能为空，最多32个字符，建议参考值：高 / 中 / 低 |
| currentNode | String | 是 | 当前状态/当前节点 | 不能为空，最多128个字符 |
| applicantId | String | 否 | 申请人ID | 可为空，最多64个字符 |
| applicantUser | String | 是 | 申请人用户名 | 不能为空，最多64个字符 |
| applicantName | String | 是 | 申请人姓名 | 不能为空，最多64个字符 |
| handlerIdList | List | 否 | 处理人ID列表 | 不为空时需要与 handlerUserList 一一对应，最多100个 |
| handlerUserList | List | 是 | 处理人用户名列表 | 不能为空，最多100个 |
| handlerNameList | List | 是 | 处理人姓名列表 | 不能为空，最多100个 |
| startTime | Date | 否 | 待办开始时间 |  |
| deadlime | Date | 否 | 待办结束时间 |  |
| systemName | String | 是 | 来源系统名称 | 不能为空，最多64个字符，例如：CNPD / IPM / PDM / PCM |
| modelId | String | 否 | 模块标识 | 例如：项目管理 |
| todoCate | String | 否 | 分类 | 例如：出差申请 |
| crtTime | Date | 否 | 待办创建时间 | 不传默认为当前时间 |
| attribute1 | String | 否 | 待办附带信息 | 用于待办完成的判断条件 |
