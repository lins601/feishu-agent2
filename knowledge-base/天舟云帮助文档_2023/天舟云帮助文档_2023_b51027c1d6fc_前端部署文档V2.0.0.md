# 1、部署背景


## 1.1 名词解释


天舟云平台：包括天舟云配置系统、天舟云预览系统、天舟云门户系统

天舟云配置系统：用于低代码应用的搭建、数据源配置、页面设计、模型设计等

天舟云预览系统：用于预览和使用搭建好的低代码应用，会根据应用id生成单独的应用系统地址，可直接供用户使用

天舟云门户系统：用于进入天舟云2.0的门户管理页面，可直接查看和管理当前租户下的所有应用，并可使用对应应用设计的页面

# 2、部署步骤


## 2.1 Apollo配置


按照👉 [文档](https://docs.cvte.com/docs/tzv16/tianzhou2-0-apollo)进行Apollo的配置并在对应环境保存发布；

❗️ 需要注意的是，Apollo中的后端服务地址，需要根据不同环境修改地址中对应的环境标识；

## 2.2 鲸云配置


按照👉 [文档](https://docs.cvte.com/docs/tzv16/tianzhou2-0-ccloud)进行鲸云应用的配置；

❗️需要注意的是，天舟云2.0使用Gitlab流水线构建，所以需要在鲸云应用中选择正确的代码分支；

## 2.3 资源版本


依照👉 [文档](https://docs.cvte.com/docs/tzv16/tianzhou2-0-resources-version)进行依赖资源的对照；

❗️需要注意的是，如果是私有化部署，强烈建议单独创建宿主资源；
