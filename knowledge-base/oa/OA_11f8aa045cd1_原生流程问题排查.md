# 原生流程问题排查

#### 范围


表单页面+流程都在OA，在页面操作时发生报错。


#### 痛点


 表单页面的定制的逻辑非常分散，难以统一查,包含以下部分
 1、配置页面定制的多个JSP
 2、标准组件本身的JSP
 3、EKP工程中的定制JS
 4、EKP工程中Java接口代码


#### 解决方案要点


 1、合并定制JS、工程代码到统一上下文
 2、引入问题排查方法的.md文件
 3、Cursor统一分析


#### 使用步骤


 1、Cursor打开”EKP”工程 文件夹
 2、配置MCP服务
 Cursor setting->tools & MCP->Add a Custom Mcp Server->编辑->填入以下内容
 `{
 "mcpServers": {
 "OA-V12-sit": {
 "url": "https://mcpsit-api.gz.cvte.cn/mcp-servers/OA-V12-sit"
 }
 }
}`
3、EKP 根目录放 “原生流程问题排查.md” (在尾部附件)
4、拖入.md文件到提示词输入框,或者把.md内容放到工程的rule中
5、写提示词：模板名称+操作步骤+报错现象
 提示词举例:


```
@oaskills/原生流程问题排查.md
流程模板: TV技术资料申请
操作步骤 ： 新建流程时填BOM号"002.006.0025132"时,
报错如下
获取物料信息错误错误原因：{
    "DESCRIPTION": "PW.M130W1.602A(25373),<47.02.0206930.001>,全电压,90VAC-264VAC,20.5V/6.2A    ",
    "HAS_NEED_SOFT": "是",
    "ID": "188741858",
    "INVENTORY_ITEM_STATUS_CODE": "中试",
    "ITEM_CODE": "002.006.0025132",
    "ITEM_ID": "4b576ff51de04655886729d287b78d9f",
    "ITEM_NAME": "电源板",
    "MIDDLE_CLASS": "TV电源板",
    "PRODUCT_MODEL": "PW.M130W1.602",
    "UOM": "PCS"
}
```

#### 附件


原生流程问题排查.md 的内容如下


```

## 一、核心工作目标
协助用户快速获取指定OA原生流程模板对应的JS代码，并分析用户的错误原因,通过标准化三步流程确保响应的准确性和高效性。

## 二、详细工作流程
### 步骤1：识别用户意图——提取模板名称
1.  接收用户输入内容（包括文字描述、提问等各类形式）；
2.  精准筛选并提取输入中明确或隐含的「OA原生流程模板名称」，忽略无关冗余信息；
3.  若用户输入中模板名称模糊、缺失或存在歧义，需先向用户确认具体模板名称，再进入后续步骤。

### 步骤2：调用工具接口——查询OA原生流程模板JS代码
1.  确认模板名称无误后，调用指定工具接口：「查询OA原生流程模板JS代码」；
2.  接口调用时需准确传入已提取的「模板名称」作为核心参数，确保接口能精准匹配对应数据；
3.  等待接口返回结果，若出现接口调用失败、超时等异常情况，需记录异常信息并反馈给用户，无法继续后续流程。

### 步骤3：输出结果——反馈接口返回内容
1.  接收接口返回的OA原生流程模板JS代码；
2.  结合本文件夹中的EKP代码分析用户的报错；
3.  输出报错原因分析结果。

## 三、注意事项
1.  全程以“准确提取模板名称-正确调用接口-如实反馈结果”为核心准则，不偏离流程要求；
2.  优先保障模板名称的准确性，这是后续接口调用和结果输出的基础；
3.  无需对接口返回的JS代码进行语法解析、优化或注释补充，仅做原样输出。
```
