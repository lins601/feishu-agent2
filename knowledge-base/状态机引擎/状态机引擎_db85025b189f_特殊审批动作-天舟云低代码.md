# 特殊审批动作-天舟云低代码

#### 接口说明


 接口名称：流程特殊审批操作
 接口地址：{app项目域名+转发路径}/form/workflow/approve/handle
 请求方式: POST JSON
 入参说明


| 参数 | 描述 | 类型 | 可否为空 | 备注说明 |
| --- | --- | --- | --- | --- |
| approveCode | 操作编码 | 字符串(String) | 不可为空 | 终审通过传:admin_finish 普通审批:pass |
| approveName | 操作名称 | 字符串(String) | 允许为空 |  |
| remark | 操作备注 | 字符串(String) | 允许为空 | 在审批记录中显示 |
| classId | 表单id | 字符串(String) | 不允许为空 | obj_class表的id |
| formInstanceId | 单据id | 字符串(String) | 不允许为空 | 业务表中的id |


请求头


| 参数 | 描述 | 类型 | 可否为空 | 备注说明 |
| --- | --- | --- | --- | --- |
| x-auth-token | 用户jwt | 字符串(String) | 可为空 | 操作记录上会作为操作人展示,和x-iac-token二选一 |
| x-iac-token | 系统iac | 字符串(String) | 可为空 | 操作记录上会作为操作人展示,和x-auth-token二选一 |
| x-app-id | 应用id | 字符串(String) | 不可为空 | obj_application表id |
| x-tenant-id | 租户id | 字符串(String) | 不允许为空 | obj_application表tenant_id |


备注: x-auth-token和x-iac-token两者必须传一个


请求参数例子


```
{
    "remark": "操作备注",
    "classId": "表单id",
    "formInstanceId": "单据id",
    "handleReason": "管理员操作",
    "approveName": "终审通过",
    "approveCode": "admin_finish"
}
```


返回信息


| 参数 | 描述 | 类型 | 可否为空 | 备注说明 |
| --- | --- | --- | --- | --- |
| status | 返回状态 | 字符串 | 不为空 | 0:成功 其他:失败 |
| success | 业务成功标识 | (boolean) | 不为空 | true:成功 false:失败 |
| result | 返回信息 | 字符串(String) | 可能为空 | 提示信息 |


示例：


```
{
    "status": "0",
    "message": "success",
    "data": {
        "success": true,
        "result": null,
        "status": null
    }
}
```
